# Contributing to Open Society ERP

Thanks for your interest in contributing! We welcome bug reports, feature requests, documentation improvements, and code contributions.

## Getting started

1. Fork the repository on GitHub and clone your fork.
2. Create a branch for your change:

```bash
git checkout -b feat/my-feature
```

3. Make small, focused commits with clear messages.
4. Run tests and linters before submitting a PR.

## Code style

- Follow the existing project coding style (TypeScript/JavaScript for backend/frontend).
- Use Prettier formatting if present; otherwise keep consistent formatting.
- Add unit tests for bug fixes and features when applicable.

## Pull request process

1. Open an issue describing the bug or feature first if it’s non-trivial.
2. Create a PR from your feature branch into `main` with a descriptive title.
3. Include screenshots or recordings for UI changes and provide reproduction steps for bugs.
4. Link related issues in the PR description.

## Commit messages

- Use present-tense, short subject lines (e.g., `fix: handle null tenantId`).
- Include a longer description in the body when necessary.

## Reporting security issues

Please do NOT open a public issue for security vulnerabilities. Instead, contact the maintainers directly via email: owner@societytracker.com.

Thanks for helping improve Open Society ERP!
